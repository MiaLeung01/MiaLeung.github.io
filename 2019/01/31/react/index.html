React 介绍

React 是什么？

React不是一个MVC框架，而是一个用于构建组件化UI的库，是一个前端界面开发工具。
React 起源于 Facebook 的内部项目，用来架设 Instagram 的网站，并于 2013 年 5 月开源。

React 设计思想

虚拟DOM：
在React的实际的使用中，需要根据不同的数据展现不同的UI,当数据变化时，
React会重新构建整个DOM树，然后将当前的DOM树和之前的比较，
得到DOM树的区别，然后仅仅把变化的部分反映到实际的浏览器UI更新上。

shouldComponentUpdate -> react性能优化非常重要的一环。组件接受新的state或者props时调用，我们可以设置在此对比前后两个props和state是否相同，如果相同则返回false阻止更新，因为相同的属性状态一定会生成相同的dom树，这样就不需要创造新的dom树和旧的dom树进行diff算法对比，节省大量性能，尤其是在dom结构复杂的时候

React diff 算法

  • react 如何计算本次更新和下一次更新的不同，做到只更新变化的节点的？有什么策略？

  • render 方法中描述的 UI 是真实的 dom 结点还是虚拟的 dom 结点呢？React 是怎么样表现这个结点的？

    diff 策略
    （1）忽略dom节点跨层级的移动
    （2）相同类型的两个组件产生的 dom 结构相似，不同类型的组件产生的 dom 结构不尽相同
    （3）对同一层级的节点，分配唯一的ID（key）进行区分

    三个优化
    （1）tree diff
    只对比较同一层级的节点。如果父节点不存在，直接删除该节点下的所有子节点，否则对比该节点下所有子节点。
    即跨层级的 dom 操作，删除原有的 dom（该节点及所有子节点）, 重新的新位置创建新的节点。
    （2）component diff

    • 组件的类型不同，直接替换整个组件。
    • 组件类型相同，按照 tree diff 比较 v-dom 树。React 允许开发者 使用 shouldComponentUpdate 决定组件是否需要更新。
      （3）element diff
    • 插入 组件的新集合中有组件旧集合中没有的类型的节点
    • 移动 组件新集合中有旧集合的类型，并且 element 可更新，可复用以前的 dom ，执行 dom 移动操作
    • 删除 组件新集合中有旧集合的类型，并且 element 不可更新；旧组件不在新集合

    对于上述三点，开发的时候

    • 尽量保持 dom 结构不变，能用样式的用样式。
    • 能保证组件不需要更新的时候，使用 shouldComponentUpdate 减少比较；尽量减少结构相同类型不同的组件（大概就是类似结构的能写出一个组件就不要写成两个）

    • 同一层级的节点，尽量添加 key ；尽量减少将最后的节点移到首部的操作。

      • 什么情况下 element 不可更新？
      • 对比的时候，是对比 v-dom 还是真实 dom。 JSX 经 babel 解析后
