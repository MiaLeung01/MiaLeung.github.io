400 的原因：

400指的的是请求无效（请求有语法问题或者不能满足请求）。400 有两类，一是 BadRequest “错误的请求”， 二是 Invalid Hostname “不存在的域名”。
其中 BadRequest 产生的原因一是前端提交数据的字段名称或者是字段类型和后台的实体类不一致，导致无法封装，二是前端后端约定的参数类型不一致，一个是 json 一个是 form。

在 SpringMVC 中的 Action 中处理前端 Ajax 请求传过来的 json 数据直接转成对应的实体类时出错会报 400 Bad Request，后端也不报错。

  1. 一般最常见的就是后台的实体类 bean 与前端传的类型不匹配，如你的 javabean 中有定义了 Date 类型和 int 类型的成员变量，导致转化器在把 json 数据转化成 bean 时不能转化。
  2. log4j的配置文件里错误将部分log打为Info级别所致
  3. 要返回 json 的却忘了加 @ResponseBody
  4. ajax 请求的连接后边忘了加参数,如url:basePath + “kscj/unbill.do?jqid=”+jqids
  5. 传了非实体bean属性的参数过来。
  6. controller指定的参数名称与前台不一致
    作者：陌上桑花开花
    原文：https://blog.csdn.net/u014079773/article/details/51865178/

今天遇到的问题原因是上述的第一个，参数需要传日期，前端传的 String, 后端定义的 Date 类型。

解决办法:

  1. 把实体类的javabean里边的类型都改成 string 类型。
  2. 在对应的实体类的对应的非字符串类型的变量的setter方法中传入string类型的，然后在里边用 SimpleDateFormat 或者Integer进行转化。
  3. 在实体类的日期属性上加@DateTimeFormat(pattern=”yyyy-MM-dd”)注解。
    详细的见原文
    作者：陌上桑花开花
    原文：https://blog.csdn.net/u014079773/article/details/51865178/

后端选择了 3 来解决这个问题，400 变成 200~
