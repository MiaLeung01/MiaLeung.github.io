兼容 IE8 的那些事

Posted on 2019-03-27 | In 兼容性

这几天在写一个填写个人信息的表单，要求兼容到 IE8，踩了几个坑，记录下：

  1. 获取URL参数

(^|&) 匹配字符串开头或者&字符，如果其后还有正则，那么必须出现在字符串开始或&字符之后

  1. 图片上传并转成 base64 格式， 兼容 IE8
    （1）文件清空删除
    选择一个图片后，删除图片，无法重新选择同一张图片。因为没有触发 input 的 change 事件，所以要重置 input 的值，或者删除 input 节点，添加一个新节点。 IE11 以下不支持文件重置值（我试了 $(node).val(‘’) 和原生的 document.getElementById(node).value = ‘’ 都不生效）
    // 兼容 IE8

    （2）base64 编码的图片提交的时候数据很大的问题
    对Post请求来说，数据是没有大小限制的，只是取决于服务器的配置。
    解决办法一是对 base64 进行压缩，二是修改服务器配置

  2. 原生js
    （1）获取节点的前一个兄弟 previousElementSibling is not supported until IE9.
    兼容：

p.s. previousElementSibling 和 previousSibling 区别，前一个是获取元素前一个元素，后一个是获取元素前一个节点（文本或元素）

（2）Object.keys(obj) IE9 及以上才支持，替代

  1. Jqurey
    -【坑】jq 版本2以上不支持 IE8
    jq 选择器：

    (1) 获取选中的项 $("input[name='" + name +"']:checked").val()

    (2) 设置选中的项 $(node).prop('checked', true)

（3）$(node).empty() 清空 node 所有子元素
（4）$(node).val() 获取/设置 value
（5）$(node).append(node1) 插入子元素
（6）$(node).addClass(‘className’); 增加 class
（7）$(node).attr(‘name’) 获取元素的属性
（8）$(node).change(func(e){}) $(node).click(func(e){})

（10）防止事件重复绑定

off()、on()方法[ie 不支持]

  1. IE8 奇葩问题

IE8 中，如果button 不加 type='button' 并放在 form 表单中
那么 ie8就会自动把它变成了 submit，点击绑定的事件将不会执行，而且会刷新页面。

解决办法是：明确加入 type='button'
