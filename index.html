  1. 什么是防抖和节流？有什么区别？如何实现？

防抖:动作绑定事件，动作发生后一定时间后触发事件，在这段时间内，如果该动作又发生，则重新等待一定时间再触发事件。

节流:动作绑定事件，动作发生后一段时间后触发事件，在这段时间内，如果动作又发生，则无视该动作，直到事件执行完后，才能重新触发。

p.s.
git没有提供远程分支删除的命令, 所以git换用push来实现
推送的命令是:
git push origin master : my-branch
意思为: 将本地的master分支推送到远程的my-branch分支, 如果把本地分支的部分去掉, 就变成:
git push origin : my-branch
这一句的意思是, 将”空内容”推送到远程的my-branch分支, 也就是把这个分支删掉了

【面题】CSS

Posted on 2019-02-17 | In 面试题
  1. CSS 盒模型
    dom 元素是由四部分组成，从内到外为 content、padding、 border 、margin
    当你用 CSS 给元素设置 width 和 height 的时候，只会应用到 content 区，如果还有 padding 和 border ，
    浏览器绘制的时候，标准的盒子宽度和高度会加上 padding 和 border。

  2. box-sizing
    content-box: 默认值，标准盒子模型。 width 与 height 只包括内容的宽和高， 不包括边框（border），内边距（padding），外边距（margin）。
    border-box: width 和 height 属性包括内容，内边距和边框，但不包括外边距。

  3. BFC (Block Fromatting Context) https://www.cnblogs.com/libin-1/p/7098468.html
    作用：

    1. 阻止 margin 融合（外边距折叠）：处于不同BFC中的元素外边距不折叠
    2. 包含浮动元素
    3. 避免文字环绕 ：BFC 内子元素贴紧父元素左外边缘

    生成 BFC：

    1. position 不是 static 或者 relative
    2. float 不为 none
    3. overflow 不是 visible
    4. display 的值为 inline-block、 table-cell、 table-caption、 flex 或者 inline-flex

  4. 层叠上下文 Stacking Context https://www.jianshu.com/p/0f88946a0746
    HTML中的三维概念，每个盒模型的位置都是三维的，分别是平面画布的 x 轴、 y 轴 和层叠的 z 轴。
    产生：

    1. HTML中的根元素本身就具有层叠上下文，称为“根层叠上下文”。
    2. 普通元素设置position属性为非static值并设置z-index属性为具体数值，产生层叠上下文。
    3. CSS3中的新属性也可以产生层叠上下文。
      层叠顺序：
      background/border
      z-index<0
      block 块级水平盒子
      float 浮动盒子
      inline/inline-block 水平盒子
  z-index auto/0
    z-index > 0

一次联调导出400的原因

Posted on 2019-02-01 | In 工作中碰到的问题

今天跟后端联调的时候遇到了个 400 Bad Request 问题，柯南模式又要开启啦。

前端关于导出（文件下载）：

  • 前端导出的方法有哪些？
    1. 使用<a> 标签
    2. window.open(url)
    3. form 表单

400 的原因：

400指的的是请求无效（请求有语法问题或者不能满足请求）。400 有两类，一是 BadRequest “错误的请求”， 二是 Invalid Hostname “不存在的域名”。
其中 BadRequest 产生的原因一是前端提交数据的字段名称或者是字段类型和后台的实体类不一致，导致无法封装，二是前端后端约定的参数类型不一致，一个是 json 一个是 form。

在 SpringMVC 中的 Action 中处理前端 Ajax 请求传过来的 json 数据直接转成对应的实体类时出错会报 400 Bad Request，后端也不报错。

  1. 一般最常见的就是后台的实体类 bean 与前端传的类型不匹配，如你的 javabean 中有定义了 Date 类型和 int 类型的成员变量，导致转化器在把 json 数据转化成 bean 时不能转化。
  2. log4j的配置文件里错误将部分log打为Info级别所致
  3. 要返回 json 的却忘了加 @ResponseBody
  4. ajax 请求的连接后边忘了加参数,如url:basePath + “kscj/unbill.do?jqid=”+jqids
  5. 传了非实体bean属性的参数过来。
  6. controller指定的参数名称与前台不一致
    作者：陌上桑花开花
    原文：https://blog.csdn.net/u014079773/article/details/51865178/

今天遇到的问题原因是上述的第一个，参数需要传日期，前端传的 String, 后端定义的 Date 类型。

解决办法:

  1. 把实体类的javabean里边的类型都改成 string 类型。
  2. 在对应的实体类的对应的非字符串类型的变量的setter方法中传入string类型的，然后在里边用 SimpleDateFormat 或者Integer进行转化。
  3. 在实体类的日期属性上加@DateTimeFormat(pattern=”yyyy-MM-dd”)注解。
    详细的见原文
    作者：陌上桑花开花
    原文：https://blog.csdn.net/u014079773/article/details/51865178/

后端选择了 3 来解决这个问题，400 变成 200~

React 介绍

React 是什么？

React不是一个MVC框架，而是一个用于构建组件化UI的库，是一个前端界面开发工具。
React 起源于 Facebook 的内部项目，用来架设 Instagram 的网站，并于 2013 年 5 月开源。

React 设计思想

虚拟DOM：
在React的实际的使用中，需要根据不同的数据展现不同的UI,当数据变化时，
React会重新构建整个DOM树，然后将当前的DOM树和之前的比较，
得到DOM树的区别，然后仅仅把变化的部分反映到实际的浏览器UI更新上。

React 生命周期

getDefaultProps
getInitialState
componentWillMount
render
componentDidMount
shouldComponentUpdate(nextProps, nextState) componentWillReceiveProps(nextProps)
componentWillUpdate 在组件将要更新时才调用，此时可以修改state
render
componentDidUpdate
componentWillUnmount

shouldComponentUpdate -> react性能优化非常重要的一环。组件接受新的state或者props时调用，我们可以设置在此对比前后两个props和state是否相同，如果相同则返回false阻止更新，因为相同的属性状态一定会生成相同的dom树，这样就不需要创造新的dom树和旧的dom树进行diff算法对比，节省大量性能，尤其是在dom结构复杂的时候

React diff 算法

  • react 如何计算本次更新和下一次更新的不同，做到只更新变化的节点的？有什么策略？

  • render 方法中描述的 UI 是真实的 dom 结点还是虚拟的 dom 结点呢？React 是怎么样表现这个结点的？

    diff 策略
    （1）忽略dom节点跨层级的移动
    （2）相同类型的两个组件产生的 dom 结构相似，不同类型的组件产生的 dom 结构不尽相同
    （3）对同一层级的节点，分配唯一的ID（key）进行区分

    三个优化
    （1）tree diff
    只对比较同一层级的节点。如果父节点不存在，直接删除该节点下的所有子节点，否则对比该节点下所有子节点。
    即跨层级的 dom 操作，删除原有的 dom（该节点及所有子节点）, 重新的新位置创建新的节点。
    （2）component diff

    • 组件的类型不同，直接替换整个组件。
    • 组件类型相同，按照 tree diff 比较 v-dom 树。React 允许开发者 使用 shouldComponentUpdate 决定组件是否需要更新。
      （3）element diff
    • 插入 组件的新集合中有组件旧集合中没有的类型的节点
    • 移动 组件新集合中有旧集合的类型，并且 element 可更新，可复用以前的 dom ，执行 dom 移动操作
    • 删除 组件新集合中有旧集合的类型，并且 element 不可更新；旧组件不在新集合

    对于上述三点，开发的时候

    • 尽量保持 dom 结构不变，能用样式的用样式。
    • 能保证组件不需要更新的时候，使用 shouldComponentUpdate 减少比较；尽量减少结构相同类型不同的组件（大概就是类似结构的能写出一个组件就不要写成两个）

    • 同一层级的节点，尽量添加 key ；尽量减少将最后的节点移到首部的操作。

      ？疑问 ？

      • 什么情况下 element 不可更新？
      • 对比的时候，是对比 v-dom 还是真实 dom。 JSX 经 babel 解析后
